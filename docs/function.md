# 功能设计

## 功能介绍

**面试模拟**

帮助求职者在求职初期能快速熟悉面试流程，进入面试者状态，以对话的方式模拟技术面试全流程。

1. 基础知识提问：基于岗位职责、岗位技能进行若干轮基础知识提问。
2. 简历针对性提问：面试官基于简历上项目经历以外的内容进行针对性提问。
3. 项目提问：面试官逐项目进行提问。

----
**面试记录**

后台功能。在面试中记录面试官和用户对话内容，面试结束后生成一份面试总结。

面试总结大纲：
1. 面试基本信息，包括公司、岗位等。
2. 面试问答记录与回答订正。
3. 面试总结报告，对面试全程进行整体评价。

----
**面试提示**

在检测到用户出现停顿时，以简短的语句在边栏上提示用户。


## 技术栈

`Python 3.11`

**前端**
使用 `Gradio` 提供的前端功能。

**后端**
- App 框架：`Gradio`
- API 框架：`FastAPI`(`RESTful`)
- LLM App 框架：`LangChain 1.0`
- 数据库：`PostgreSQL 15`, `SQLAlchemy`


## 服务端

服务端功能够用即可。简单起见，对服务端进行以下简化：
- 管理功能封装成 CLI 工具。客户端调用的功能全部封装成 API。
- 不设用户管理系统，无需登录。
- 不设简历筛查、面试邀约等复杂功能。
- 岗位职责与要求仅包括和技术面试相关内容，不包含学历、工作年限等和技术、项目无关内容。
- 面试官是一个语言模型，和面试者以文字的形式交互。

----
**岗位模块**

- 增删岗位。一个岗位由一个 JSON 文件配置，包括：面试官、岗位职责、岗位要求、岗位题库。
- 查看岗位。包括查看当前已配置岗位和查看某个岗位的配置文件。
- 题库模块
	- 配置岗位题库。针对面试全流程中的各个阶段生成对应的题目。
	- 题库创建。由 LLM 预先生成某一个领域的题库供面试使用。（例如：Python 题库）一道题由题干、参考答案、评分细则构成。
	- 题目抽取。指定一个领域题库和子领域，抽取一个问题。

----
**智能体模块**
- 面试官配置。设置面试官的 system prompt、使用的模型。
- 智能体配置。配置简历预处理、面试中用到的所有智能体。
- 模型管理模块。管理本地模型、云推理服务。

----
**简历模块**

- 简历接收。接收简历 Markdown 文档。
- <进阶> PDF 格式简历转 Markdown。
- 简历结构化提取模块。结构化提取简历信息，预处理成后续面试所需样式。

----
 **面试系统**
 
- 创建面试。用户发起面试请求，在面试准备就绪（简历预处理、面试官就绪）后创建面试。向用户端提供面试查询 API。
- 面试工作流。从开始面试到面试结束生成记录和总结。
- 语音识别模块。语音转文字，<进阶> 文字转语音。
- 回答评价模块。对比参考答案和评分细则给出评分，评估超出评分细则的回答内容正确性，<进阶> 评估作答流畅度。
- <进阶> 面试提示模块。

----
**系统功能**

- 数据库模块。数据库相关功能。
- 日志模块。
- API。所有客户端调用的功能封装成 RESTful API。
- Gradio App。服务端的管理、配置功能通过 Gradio 实现。


## 客户端

客户端为一个 Gradio App，调用服务端提供的 API。

客户端功能：
- 查看当前支持面试的岗位。
- 简历上传。上传 Markdown 格式简历文件。
- 在线面试。以一个 对话式 Chatbot 的形式进行面试。
- <进阶> 面试提示功能开关。
- 查看面试总结报告。
- 查看历史面试记录。
