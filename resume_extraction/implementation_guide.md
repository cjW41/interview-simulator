# 简历结构化提取智能体实现指南

## 1. 项目概述

本项目旨在实现一个简历结构化提取智能体，能够从各种格式的简历中提取结构化信息，包括个人基本信息、教育背景、工作经历、技能等。

## 2. 技术架构

### 2.1 目录结构

```
resume_extraction/

├── requirements.txt       # 依赖项
├── src/
```

### 2.2 核心组件

1. **Extractor**：核心提取器，协调不同解析器和处理流程
2. **Parsers**：不同格式的解析器，处理不同类型的简历文件
3. **Models**：数据模型，定义结构化输出格式
4. **Utils**：工具函数，提供文本处理等功能
5. **Validators**：验证器，确保提取数据的准确性

## 3. 工作流程

### 3.1 提取流程

1. **输入处理**：接收简历文件或文本
2. **格式检测**：检测简历格式（Markdown、文本、PDF等）
3. **解析处理**：使用对应解析器解析简历内容
4. **信息提取**：从解析结果中提取结构化信息
5. **数据验证**：验证提取信息的完整性和准确性
6. **输出结果**：返回结构化的简历信息

### 3.2 模型配置

### 3.2.1 核心模型（兼容service_end/data/model.py）

- **WorkExperience**：一个岗位的工作经历
  - job: 工作岗位信息
  - year: 工作年限
  - duty: 工作经历描述

- **CVBasicInfo**：简历基本信息
  - name: 姓名
  - work_year: 工作年限
  - education_experience: 教育经历列表
  - work_experience: 工作经历列表

- **CVModel**：简历模型
  - title: cv 的唯一标识
  - basic_info: 基本信息
  - skills: 技能列表
  - project_experience: 项目经验列表

### 3.2.2 扩展模型（内部使用）

- **个人基本信息**：姓名、性别、年龄、联系方式、地址
- **教育背景**：学校、专业、学位、毕业时间、GPA
- **工作经历**：公司名称、职位、工作时间、工作描述
- **项目经验**：项目名称、角色、项目时间、项目描述、技术栈
- **技能**：技术技能、语言技能、软技能
- **证书**：证书名称、颁发机构、获得时间
- **其他信息**：自我评价、兴趣爱好等

## 4. 实现方案

### 4.1 核心提取器

- 实现`ResumeExtractor`类，提供统一的提取接口
- 支持多种输入格式：文件路径、文本内容
- 自动检测并使用合适的解析器
- 协调整个提取流程
- 提供`to_cv_model`方法，将提取结果转换为符合service_end/data/model.py格式的CVModel
- 实现工作年限计算功能

### 4.2 解析器实现

- **Markdown解析器**：处理Markdown格式的简历
- **文本解析器**：处理纯文本格式的简历
- **PDF解析器**：处理PDF格式的简历（可选，需要额外依赖）

### 4.3 数据模型

使用Pydantic V2定义数据模型，确保数据结构的一致性和验证。

### 4.4 工具函数

- 文本预处理：去除噪声、标准化格式
- 信息提取：使用正则表达式和规则提取关键信息
- 数据清洗：处理提取结果，确保数据质量

### 4.5 验证器

- 验证提取数据的完整性
- 检查数据格式的正确性
- 提供数据质量评估

## 5. 集成方案

- 提供与现有系统的集成接口
- 支持异步调用
- 提供批处理能力

## 6. 测试策略

- 单元测试：测试各个组件的功能
- 集成测试：测试整个提取流程
- 性能测试：测试提取速度和资源消耗
- 准确性测试：测试提取结果的准确性

## 7. 未来扩展

- 支持更多简历格式

- 支持多语言简历
